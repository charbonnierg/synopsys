Subscribers

Now that we have defined a channel, let's add a subscriber into our application in order to process published messages.

Definitionâ€‹

What is a subscriber ?

A subscriber is a component that listens for messages on a particual channel and react to it.

danger

A subscriber can be used to received messages with at most once delivery. It does not guarantee that no messages are lost or than messages are processed in order. If at least once or exactly once delivery is required, use consumers instead.

Even though at most delivery may lead to message losses, it is still useful in various scenarios. For example, when only most up-to-date data is used, and data is updated frequently, at most once delivery is a good solution.

Create a subscriberâ€‹

Recap

Channel has been defined using the following python code:

MEASURES = create_channel(
    name="measures",
    address="locations/{location}/devices/{device_id}/sensors/{sensor_id}/measure",
    parameters=MeasureParameters,
    schema=MeasureData,
)

Define a handlerâ€‹

  • Python SDK
from synopsys import Message
from .definitions import MeasureData, MeasureParameters


async def handler(msg: Message[MeasureData, MeasureParameters]) -> None:
    """Process messages delivered to subscriber."""
    params = msg.parameters
    measurement = msg.data
    print(f"[{params.location}] - Device {params.device_id} - Sensor {params.sensor_id}: {measurement.value}")
Handler signature

A python subscriber handler must accept a Message[DataT, ParamsT] as arguments, where DataT is the payload schema of susbcriber channel, and ParamsT is the parameters schema of subscriber channel.

If subscriber requires dependencies, such as a database connection, or an HTTP client, handlers can be created as callable classes:

class SomeDatabaseClient:
    """A fake database client."""
    pass


class HandlerWithDependencies:
    def __init__(self, database: SomeDatabaseClient) -> None:
        """Create a new handler."""
        self.db = database

    async def __call__(self, msg: Message[MeasureData, MeasureParameters]) -> None:
        """Execute handler, which relies on a database connection."""
        # Do something with the message and the database
        # await self.db.update(...)

Create subscriberâ€‹

  • Python SDK
from synopsys import Subscriber
from .channels import MEASUREMENTS


MEASUREMENTS_PROCESSOR = Subscriber(
    name="process-measurements",
    channel=MEASUREMENTS,
    handler=handler,
)

Standalone usageâ€‹

  • Python SDK
from synopsys import create_bus
from .actors import MEASUREMENTS_PROCESSOR


async def main() -> None:
    async with create_bus("nats://localhost:4222") as bus:
        await bus.register_subscriber(MEASUREMENTS_PROCESSOR)


if __name__ == "__main__":
   # Run the main function and block until shutdown
   import anyio
   anyio.run(main)

The function above will run until bus or channel is closed. When using PubSub backend with support for automatic reconnections (such as NATS), the function above runs forever (I.E, until external cancellation or failure).

Dynamic subscribersâ€‹

  • Python SDK
from synopsys import create_bus
from .actors import MEASUREMENTS_PROCESSOR


async def main() -> None:
    async with create_bus("nats://localhost:4222") as bus:
        # Register subscriber when needed
        subscriber = await bus.register_subscriber(MEASUREMENTS_PROCESSOR)
        # Stop suscriber when needed
        await subscriber.stop()


if __name__ == "__main__":
   # Run the main function and block until shutdown
   import anyio
   anyio.run(main)
