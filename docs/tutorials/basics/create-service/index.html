Services

The channel defined previously does not define a reply schema. Let's create a new channel, with a reply schema, in order to create a service.

Definition​

What is a service ?

A service is a specific kind of subscriber, which listens messages on a particual channel and send back replies to message producer.

Just like any other subscribers, a service is associated with a particular channel, however services channels must always define a reply schema.

Messages received by services are called requests, and messages sent back by services are called replies.

Comparison with HTTP

Services are very similar to HTTP endpoints. Just like them, they receive requests and send back responses. However, replies do not have concept such as status code, it's up to application developers to describe success or failure through payload or headers.

Fields​

A service defines the following fields:

DescriptionOptionalDefault
NameThe name of the service. In practice, this is the name of a command or a query.🧨🧨
ChannelThe channel delivering messages to the service. A service channel is annotated using DataT, ParamsT, ReplyT.🧨🧨
HandlerA function to execute each time a request is received. This function must return either the reply to send back to message producer, or some object of type ReplyT which whill be sent back within reply. In python, handlers must be coroutine functions returning a Reply object.🧨🧨
QueueWhen several services are started on the same channel, each message is delivered to a single subscriber within each queue. When no queue is specified, subscriber receives all messages.✅null

Create a service​

Let's create a very simple service which returns the allowed list of locations. It could be used by clients wishing to publish messages in order to verify that their configured location is valid.

It should not accept paremeters or request body, and it should return a list of locations as strings.

Create a channel​

  • Python SDK
import typing as t
from synopsys import create_channel


LOCATIONS = create_channel(
    name="query-locations",
    address="query/locations",
    reply_schema=t.List[str],
)

Define a handler​

  • Python SDK
from synopsys import Request


async def handler(msg: Request[None, None, t.List[str]]) -> t.List[str]:
    """Returns a list of allowed locations for sensors measurements."""
    return ["paris", "grenoble", "test-location"]
Handler signature

A python service handler must accept a Request[DataT, ParamsT, ReplyT] as arguments, where:

  • DataT is the payload schema of service channel
  • ParamsT is the parameters schema of service channel.
  • ReplyT is the reply schema of service channel.

In the example above, the handler accepts a Request[None, None, t.List[str]] because channel does not define a payload schema and does not define parameters. Last annotation is t.List[str] as defined in channel reply schema. Like subscribers, if a service requires dependencies, such as a database connection, or an HTTP client, handlers can be created as callable classes:

class SomeDatabaseClient:
    """A fake database client."""
    pass


class ServiceWithDependency:
    def __init__(self, database: SomeDatabaseClient) -> None:
        """Create a new handler."""
        self.db = database

    async def __call__(self, msg: Request[None, None, t.List[str]]) -> t.List[str]:
        """Execute handler, which relies on a database connection."""
        # Query the database and return a list of strings
        # await self.db.query(...)
        return ["test-location"]

Create the service​

  • Python SDK
from synopsys import Service
from .channels import LOCATIONS


LOCATIONS_SERVICE = Subscriber(
    name="locations-service",
    channel=LOCATIONS,
    handler=handler,
)

Standalone usage​

  • Python SDK
from synopsys import create_bus
from .actors import LOCATIONS_SERVICE


async def main() -> None:
    async with create_bus("nats://localhost:4222") as bus:
        await bus.register_service(LOCATIONS_SERVICE)


if __name__ == "__main__":
   # Run the main function and block until shutdown
   import anyio
   anyio.run(main)

The function above will run until bus or channel is closed. When using PubSub backend with support for automatic reconnections (such as NATS), the function above runs forever (I.E, until external cancellation or failure).

Dynamic services​

  • Python SDK
from synopsys import create_bus
from .actors import LOCATIONS_SERVICE


async def main() -> None:
    async with create_bus("nats://localhost:4222") as bus:
        service = await bus.register_service(LOCATIONS_SERVICE)
        # Stop service when needed
        await service.stop()


if __name__ == "__main__":
   # Run the main function and block until shutdown
   import anyio
   anyio.run(main)
