Consumers

Previously, we created a susbcriber, however, as it was discussed, subscribers do not guarantee that messages are always received, and do not guarantee that messages are processed in order.

Consumers are used when processing messages in order is a requirement, or when all messages must be delivered at least once.

Definition​

What is a consumer

A consumer is a stateful view of a channel. It acts as interface for clients to consume a subset of messages published in a channel and will keep track of which messages were delivered and acknowledged.

Unlike subscribers or services, messages delivered by consumers must be acknowledged before a configured timeout. If messages are not acknowledged before timeout, they are redelivered.

Messages received by consumers are called Orders.

Types of messages until now
  • Subscribers receive Messages
  • Services receive Requests
  • Consumers receive Orders

Each job can be acknowledged in 3 different ways:

  • ACK: Indicate that order was processed and MUST NOT be redelivered.
  • NACK: Indicate that order was not processed and MUST be redelivered.
  • TERM: Indicate that order was not processed and MUST NOT be redelivered.
  • INPROGRESS: Indicate that order is being processed and SHOULD NOT be redelivered yet.

Fields​

A consumer defines the following fields:

DescriptionOptionalDefault
NameThe name of the consumer. In practice, this is the name of an operation.🧨🧨
ChannelThe channel delivering orders to the consumer.🧨🧨
HandlerA function to execute each time an order is received. This function is responsible for acknowledging the order. In python, handlers must be coroutine functions returning None.🧨🧨
Max Ack WaitingDuration to wait for an acknowledgement after an order has been delivered before redelivering the message✅30s
Max Ack PendingWhen maximum number of pending acknowledgement is reached, orders are not delivered until an already delivered order is acknowledged or expired. Set this value to 1 (the default) in order to process messages in order.✅1
Max DeliverMaximum number of times an order can be delivered before being discarded. Set this value to 0 in order to always redeliver messages.✅5

Create a consumer​

Let's create a consumer persisting measurements data. Because a channel already defines the measurement messages, it is not needed to create a new channel.

tip

Subscribers and consumers are not exclusive. It's possible to create as many subscribers and as many consumers as desired on a single channel.

Recap

Channel has been defined using the following python code:

MEASURES = create_channel(
    name="measures",
    address="locations/{location}/devices/{device_id}/sensors/{sensor_id}/measure",
    parameters=MeasureParameters,
    schema=MeasureData,
)

Define a handler​

  • Python SDK
from synopsys import Job
from .definitions import MeasureData, MeasureParameters


async def handler(job: Job[MeasureData, MeasureParameters]) -> None:
    """Process messages delivered to subscriber."""
    params = job.parameters
    measurement = job.data
    print(f"[{params.location}] - Device {params.device_id} - Sensor {params.sensor_id}: {measurement.value}")
    # Let's imagine that we persist the data here
    # ...
    # Acknowledge the job
    await job.ack()
Handler signature

A python consumer handler must accept a Job[DataT, ParamsT] as arguments, where DataT is the payload schema of consumer channel, and ParamsT is the parameters schema of consumer channel.

If consumer requires dependencies, such as a database connection, or an HTTP client, handlers can be created as callable classes:

class SomeDatabaseClient:
    """A fake database client."""
    pass


class HandlerWithDependencies:
    def __init__(self, database: SomeDatabaseClient) -> None:
        """Create a new handler."""
        self.db = database

    async def __call__(self, job: Job[MeasureData, MeasureParameters]) -> None:
        """Execute handler, which relies on a database connection."""
        # Do something with the job data and the database
        # await self.db.update(...)

Create consumer​

  • Python SDK
from synopsys import Consumer
from .channels import MEASUREMENTS


MEASUREMENTS_CONSUMER = Consumer(
    name="persist-measurements",
    channel=MEASUREMENTS,
    handler=handler,
    max_ack_waiting=1,
    max_deliver=10,
)

Standalone usage​

  • Python SDK
from synopsys import create_bus
from .actors import MEASUREMENTS_CONSUMER


async def run() -> None:
    async with create_bus("nats://localhost:4222") as bus:
        await bus.register_consumer(MEASUREMENTS_CONSUMER)

The function above will run until bus or channel is closed. When using PubSub backend with support for automatic reconnections (such as NATS), the function above runs forever (I.E, until external cancellation or failure).

Dynamic consumers​

  • Python SDK
from synopsys import create_bus
from .actors import MEASUREMENTS_CONSUMER

async def main() -> None:
    async with create_bus("nats://localhost:4222") as bus:
        consumer = await bus.register_consumer(MEASUREMENTS_CONSUMER)
        # Do something
        # ...
        # Stop the consumer
        await consumer.stop()
